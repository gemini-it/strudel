# Métadonnées musicales

Vous pouvez optionnellement ajouter des métadonnées musicales à votre code Strudel, en utilisant des balises dans les commentaires :

```js
// @title Hey Hoo
// @by Sam Tagada
// @license CC BY-NC-SA
```

Comme d'autres commentaires, ceux-ci sont ignorés par Strudel, mais peuvent être utilisés par d'autres outils pour récupérer des informations sur la musique.

## Syntaxe alternative

Vous pouvez également utiliser des blocs de commentaires :

```js
/*
@title Hey Hoo
@by Sam Tagada
@license CC BY-NC-SA
*/
```

Ou définir plusieurs balises sur une ligne :

```js
// @title Hey Hoo @by Sam Tagada @license CC BY-NC-SA
```

La balise `title` a une syntaxe alternative utilisant des guillemets (doit être définie tout au début) :

```js
// "Hey Hoo" @by Sam Tagada
```

## Liste des balises

Les balises disponibles sont :

- `@title` : titre de la musique
- `@by` : auteur(s) de la musique, séparés par une virgule, éventuellement suivi d'un lien dans `<>` (ex : `@by John Doe <https://example.com>`)
- `@license` : licence(s) de la musique, par exemple CC BY-NC-SA. Pas sûr ? [Choisissez une licence Creative Commons ici](https://creativecommons.org/choose/)
- `@details` : quelques informations supplémentaires sur la musique
- `@url` : page(s) web liée(s) à la musique (dépôt git, lien SoundCloud, etc.)
- `@genre` : genre(s) musical(aux) (pop, jazz, etc)
- `@album` : nom de l'album musical

## Valeurs multiples

Certaines balises acceptent plusieurs valeurs, en utilisant la virgule ou le séparateur de nouvelle ligne, ou en dupliquant la balise :

```js
/*
@by Sam Tagada
    Jimmy
@genre pop, jazz
@url https://example.com
@url https://example.org
*/
```

Vous pouvez également ajouter des préfixes optionnels et utiliser des balises où vous le souhaitez :

```js
/*
song @by Sam Tagada
samples @by Jimmy
*/
...
note("a3 c#4 e4 a4") // @by Sandy
```
