# Utiliser Strudel hors ligne

Vous pouvez utiliser Strudel même sans réseau ! Lorsque vous visitez pour la première fois le [Strudel REPL](https://strudel.cc/),
votre navigateur téléchargera toute l'application web y compris la documentation.
Lorsque le téléchargement est terminé (&lt;1MB), vous pouvez visiter le site web même hors ligne,
obtenant le site web téléchargé au lieu de celui en ligne.

Lorsque le site est mis à jour, votre navigateur téléchargera cette mise à jour lors de la prochaine visite en ligne.
Lorsqu'une mise à jour est disponible, le site s'actualisera une fois le téléchargement terminé.

Cela fonctionne parce que Strudel est implémenté comme une application web progressive (utilisant [Vite PWA](https://vite-pwa-org.netlify.app/)).

## Échantillons

Bien que le navigateur télécharge l'application elle-même, les échantillons ne sont téléchargés que lorsque vous les utilisez activement.
Donc, pour vous assurer qu'un ensemble spécifique d'échantillons est disponible hors ligne, utilisez-les simplement.
De plus, seuls les échantillons de ces domaines seront mis en cache pour une utilisation hors ligne :

- `https://raw.githubusercontent.com/*` pour les échantillons téléchargés sur GitHub
- `https://freesound.org/*` / `https://cdn.freesound.org/*` pour freesound
- `https://shabda.ndre.gr/.*` pour shabda

## Inspecter / Effacer le cache

Vous pouvez voir tous les fichiers mis en cache dans votre navigateur.

### Firefox

- Ouvrez les Outils de développement (`Outils > Développement Web > Outils de développement Web`)
- allez dans l'onglet `Stockage` et développez `Cache Storage > https://strudel.cc`.
- ou allez dans l'onglet `Application` et visualisez les dernières mises à jour dans `Service Workers`

### Navigateurs basés sur Chromium

- Ouvrez les Outils de développement (`Clic droit > Inspecter`)
- allez dans l'onglet `Application`
- visualisez les fichiers téléchargés sous `Cache > Cache Storage`
- visualisez les dernières mises à jour dans `Service Workers`

## Application autonome Strudel

Vous pouvez également installer Strudel comme application autonome sur la plupart des appareils.
Une application autonome a sa propre icône de bureau / d'écran d'accueil et se lance dans une fenêtre séparée,
sans l'interface utilisateur du navigateur.

<figure>
  ![Strudel sur MacOS](/pwa/strudel-macos.png)
  <figcaption>Strudel sur MacOS</figcaption>
</figure>

### Bureau

Avec un navigateur basé sur Chromium :

1. allez sur le [Strudel REPL](https://strudel.cc).
2. à droite de la barre d'adresse, cliquez sur `installer Strudel REPL`
3. le REPL devrait maintenant s'exécuter comme une application Chromium autonome

Sans navigateur basé sur Chromium, vous pouvez utiliser [nativefier](https://github.com/nativefier/nativefier) pour générer une application de bureau :

1. assurez-vous d'avoir NodeJS installé
2. exécutez `npx nativefier strudel.cc`

<figure>
  ![Strudel sur Linux](/pwa/strudel-linux.png)
  <figcaption>Strudel sur Linux</figcaption>
</figure>

### iOS

1. ouvrez le [Strudel REPL](https://strudel.cc/) dans Safari
2. appuyez sur l'icône de partage et l'onglet `Ajouter à l'écran d'accueil`
3. Vous devriez maintenant avoir une icône d'application Strudel qui ouvre le REPL en plein écran

### Android

1. ouvrez le [Strudel REPL](https://strudel.cc/)
2. Appuyez sur le bouton d'installation en bas
