# Mondo Notation

"Mondo Notation" est un nouveau type de notation similaire à la [Mini Notation](/learn/mini-notation/), mais avec suffisamment de capacités pour en faire un langage de pattern autonome.
Voici un exemple :

<MiniRepl
  mondo
  client:idle
  tune={`$ note (c2 # euclid <3 6 3> <8 16>) # *2 
  # s "sine" # add (note [0 <12 24>]*2)
  # dec(sine # range .2 2) 
  # room .5
  # lpf (sine/3 # range 120 400)
  # lpenv (rand # range .5 4)
  # lpq (perlin # range 5 12 # * 2)
  # dist 1 # fm 4 # fmh 5.01 # fmdecay <.1 .2>
  # postgain .6 # delay .1 # clip 5

$ s [bd bd bd bd] # bank tr909 # clip .5

# ply <1 [1 [2 4]]>

$ s oh\*4 # press # bank tr909 # speed.8

# dec (<.02 .05>\*2 # add (saw/8 # range 0 1))

`}
/>

## Mondo dans le REPL

Pour l'instant, vous ne pouvez utiliser mondo dans le REPL que comme ceci :

<MiniRepl client:idle tune={'mondo`s hh*8`'} />

Le reste de ce site utilisera uniquement la notation mondo elle-même.
À l'avenir, le REPL pourrait avoir un moyen d'utiliser la notation mondo directement.

## Appel de fonctions

Par rapport à la Mini Notation, la caractéristique la plus notable de la Mondo Notation est la capacité d'appeler des fonctions en utilisant des parenthèses rondes :

<MiniRepl mondo client:idle tune={`(s hh*8)`} />

Le premier élément à l'intérieur des parenthèses est le nom de la fonction. En JS, cela ressemblerait à :

<MiniRepl client:idle tune={`s("hh*8")`} />

Les parenthèses les plus externes ne sont pas nécessaires, nous pouvons donc les supprimer :

<MiniRepl mondo client:idle tune={`s hh*8`} />

## Fonctionnalités de la Mini Notation

En plus de l'appel de fonction avec des parenthèses rondes, la Mondo Notation a beaucoup en commun avec la Mini Notation :

### Crochets

- `[]` pour les séquences d'un cycle
- `<>` pour les séquences multi-cycles
- `{}` pour les séquences par pas (plus sur cela plus tard)

### Opérateurs infixes

- \* => [fast](/learn/time-modifiers/#fast)
- / => [slow](/learn/time-modifiers/#slow)
- ! => [extend](/learn/stepwise/#extend)
- @ => [expand](/learn/stepwise/#expand)
- % => [pace](/learn/stepwise/#pace)
- ? => [degradeBy](/learn/random-modifiers/#degradeby) (nécessite actuellement l'opérande droit)
- : => tail (crée une liste)
- .. => range (entre les nombres)
- , => [stack](/learn/factories/#stack)
- | => [chooseIn](/learn/random-modifiers/#choose)

### Exemple

<MiniRepl
  mondo
  client:idle
  tune={`note <
[e5 [b4 c5] d5 [c5 b4]]
[a4 [a4 c5] e5 [d5 c5]]
[b4 [~ c5] d5 e5]
[c5 a4 a4 ~]
[[~ d5] [~ f5] a5 [g5 f5]]
[e5 [~ c5] e5 [d5 c5]]
[b4 [b4 c5] d5 e5]
[c5 a4 a4 ~]
>`}
/>

## Chaînage de fonctions

Similaire à comment "." fonctionne en JavaScript (JS), nous pouvons chaîner les appels de fonctions avec l'opérateur "#" :

<MiniRepl
  mondo
  client:idle
  tune={`n <0 2 4 [3 1] -1>*4 
 # scale C4:minor 
 # jux rev 
 # dec .2
 # delay .5`}
/>

Voici la même chose écrite en JS :

<MiniRepl
  client:idle
  tune={`n("<0 2 4 [3 1] -1>*4")
 .scale("C4:minor")
 .jux(rev)
 .dec(.2)
 .delay(.5)`}
/>

### Chaînage de fonctions localement

Une fonction peut être appliquée à un seul élément en l'enveloppant dans des parenthèses rondes :

<MiniRepl mondo client:idle tune={`s [bd hh bd (cp # delay .6)] # bank tr909`} />

dans ce cas, `delay .6` ne sera appliqué qu'à `cp`. comparez ceci avec la version JS :

<MiniRepl client:idle tune={`s(seq("bd", "hh", "bd", "cp".delay(.6))).bank('tr909')`} />

ici nous pouvons voir combien nous pouvons économiser quand il n'y a pas de frontière entre la mini notation et les appels de fonctions !

### Chaînage d'opérateurs infixes

Les opérateurs infixes existent comme des fonctions régulières, donc ils peuvent être chaînés également :

<MiniRepl client:idle mondo tune={`s [bd hh] # bank tr909 # *2`} />

Dans ce cas, le \*2 sera appliqué à tout le pattern.

### Fonctions Lambda

Certaines fonctions dans Strudel attendent une fonction en entrée, par exemple :

<MiniRepl client:idle tune={`n("0 .. 7").scale("C:minor").sometimes(x=>x.dec(.1))`} />

dans mondo, le `x=>x.` peut être raccourci en :

<MiniRepl client:idle mondo tune={`n 0..7 # scale C:minor # sometimes (# dec .1)`} />

le chaînage fonctionne comme prévu :

<MiniRepl client:idle mondo tune={`n 0..7 # scale C:minor # sometimes (# dec .1 # jux rev)`} />

## Chaînes de caractères

Vous pouvez utiliser des "guillemets doubles" et des 'guillemets simples' pour obtenir une chaîne :

<MiniRepl client:idle mondo tune={`n 0..7 # scale 'C minor'`} />

## Patterns multiples

Le signe `$` peut être utilisé pour séparer plusieurs patterns :

<MiniRepl
  client:idle
  mondo
  tune={`$ s [bd rim [~ bd] rim] # bank tr707
$ chord <Dm9!3 Db7> # voicing
  # struct[x ~ ~ x ~ x ~ ~] # delay .5`}
/>

Le signe `$` est un alias pour `,` donc il créera un stack en coulisses.

## variables

en utilisant le mot-clé `def`, vous pouvez définir des variables :

<MiniRepl
  client:idle
  mondo
  tune={`
$ def melody [0 1 2 3]
$ n melody # scale C:minor
`}
/>
