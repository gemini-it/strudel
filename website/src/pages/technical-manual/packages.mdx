# Packages Strudel

Le [dépôt strudel](https://codeberg.org/uzu/strudel) est organisé comme un monorepo, contenant plusieurs packages npm.
Le but des multiples packages est de

- organiser la base de code en morceaux plus modulaires et encapsulés
- pouvoir désactiver certaines fonctionnalités
- garder les dépendances des packages core petites

## Vue d'ensemble

[Voir les derniers packages publiés sur npm](https://www.npmjs.com/search?q=%40strudel).
Voici un aperçu de tous les packages :

### Packages Parapluie

Ces packages vous donnent un point de départ tout compris, et probablement ce que vous voudriez utiliser dans votre projet :

- [repl](https://codeberg.org/uzu/strudel/src/branch/main/packages/repl) : Le REPL Strudel comme composant web.
- [web](https://codeberg.org/uzu/strudel/src/branch/main/packages/web) : Bibliothèque Strudel pour le navigateur, sans UI.

Pour en savoir plus sur ces deux packages, lisez [Utiliser Strudel dans votre Projet](/technical-manual/project-start)

### Packages Essentiels

Ces packages sont les plus essentiels. Vous voudrez peut-être tous les utiliser si vous utilisez strudel dans votre projet :

- [core](https://codeberg.org/uzu/strudel/src/branch/main/packages/core#strudelcore) : moteur de patterns tidal avec primitives core
- [mini](https://codeberg.org/uzu/strudel/src/branch/main/packages/mini#strudelmini) : parseur de mini notation + liaisons core
- [transpiler](https://codeberg.org/uzu/strudel/src/branch/main/packages/transpiler#strudeltranspiler) : transpileur de code utilisateur. sucre syntaxique + coloration

### Extensions de Langage

Ces packages étendent le langage de patterns par des fonctions spécifiques

- [tonal](https://codeberg.org/uzu/strudel/src/branch/main/packages/tonal) : fonctions tonales pour gammes et accords
- [xen](https://codeberg.org/uzu/strudel/src/branch/main/packages/xen) : fonctions microtonales / xénharmoniques

### Sorties

Ces packages fournissent des liaisons pour différentes façons de sortir les patterns strudel :

- [webaudio](https://codeberg.org/uzu/strudel/src/branch/main/packages/webaudio#strudelwebaudio) : la sortie webaudio par défaut
- [osc](https://codeberg.org/uzu/strudel/src/branch/main/packages/osc#strudelosc) : liaisons pour communiquer via OSC
- [midi](https://codeberg.org/uzu/strudel/src/branch/main/packages/midi#strudelmidi) : liaisons webmidi
- [csound](https://codeberg.org/uzu/strudel/src/branch/main/packages/csound#strudelcsound) : liaisons csound
- [soundfonts](https://codeberg.org/uzu/strudel/src/branch/main/packages/serial#strudelsoundfonts) : support Soundfont
- [serial](https://codeberg.org/uzu/strudel/src/branch/main/packages/serial#strudelserial) : liaisons webserial

### Autres

- [embed](https://codeberg.org/uzu/strudel/src/branch/main/packages/embed#strudelembed) : composant web REPL intégrable

### Plus Maintenus

- [react](https://www.npmjs.com/package/@strudel.cycles/react) : hooks et composants react pour strudel
- [eval](https://www.npmjs.com/package/@strudel.cycles/eval) : ancien transpileur de code
- [tone](https://www.npmjs.com/package/@strudel.cycles/tone) : liaisons pour instruments et effets Tone.js
- [webdirt](https://www.npmjs.com/package/@strudel.cycles/webdirt) : liaisons webdirt, remplacées par le package webaudio
- tous les packages `@strudel.cycles/*` ont été renommés en `@strudel/*` depuis la version 0.10.0.

## Outils

- [lerna](https://lerna.js.org/) pour l'incrémentation des versions
- voir CONTRIBUTING.md pour plus d'informations
